blog恢复使用

Jul 27, 2020 | 阅读

这一段时间没见

我主要做了两件事：

1.刷 leetcode 题

2.整理 GitHub 的各种历史遗留问题


那我们就从这两件事情说开去


刷 leetcode 题


leetcode 刷题已经持续了能有一个多月，第一开始的初衷是挑战自己的编程能力。

后来我发现求职时候 leetcode 成绩在简历中十分重要，便索性进行了下去。

在刷题实践中才能发现自己曾经有多么弱。

我这一个多月里学演练了哈希表、堆栈、dfs，感到之前对算法的理解只是停留在书本，

真正使用起来，落地实践的时候就抓瞎。

希望我能尽快刷题到一定数量，摆脱菜鸡的身份。

另外每周的周赛我也有参加，排名就在中间不上不下，看以后的发挥吧。

但愿自己能成长得快一点，早日找到心仪的工作。


整理 GitHub 的各种历史遗留问题


1. 删除 fork 垃圾堆


之前刚接触 GitHub 的时候对于这个平台的操作并不了解，

对代码的开发也不怎么了解，

天天 fork 别人的东西当作收藏，

或者找到现成的模板，改改参数当自己的东西用。

我猜想大家刚接触编程的时候都会这样做吧，

改个名称大小颜色之类的，只要能跑起来，

就觉得 “哇！我会编程啦！”

直到今年开始从零做脚本时才发现自己不懂的东西太多了，

而 “fork 过就是学会了” 这种想法是完全没有帮助到我的。

于是上个月整顿了账号，只保留我从头做的和一些文档教程，

删除 fork 垃圾堆之后神清气爽，

我要真正切实地去写码，锻炼我的思维和技艺。


2. 重启 blog 与整理知识


这个 blog 已经一年多没有更新过了，

当年使用它只是单纯地为了装比。

blog 的框架是现成的，外观主题是现成的，评论系统也是现成的。

这样一个缝合怪放在这里未免大材小用了，

接下来我要让这个网站真正起到它的作用，

实现日常的记录和积累。

另外我也开了另一个 repo 来放整理好的编程知识，

做成一个类似教程的东西给新手练习，

当作是我为曾经的自己准备的吧：

脏脏的编程课


3. 首页介绍文档


这是 Github 新出的功能，如果建立一个与自己 ID 相同的 repo，

那么这个 repo 的 readme.md 将在自己的首页进行显示。



而这个功能配合 Github Actions 里面的自动更新功能，

将有可能实现动态首页的效果。

在接下来的一段时间里我将着手试试看如果实现这样炫酷的首页效果。

而成果我一定会发教程来炫耀的，这个放心好了。



那么今天就先絮叨到这里，后面还会经常更新的。

人生苦短，祝我们未来都能如愿。
